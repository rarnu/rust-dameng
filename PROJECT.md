# Rust Dameng Database Driver (Rust 达梦数据库驱动)

## 项目要求

参考已提供的Python版本的达梦数据库驱动的源码，开发 rust 版本的达梦数据库驱动。

已知信息:
> Python 的驱动源码位于本地 ```./dmPython```。
> Rust 版本的驱动需要完全实现 rust 下的 sqlx 标准，以异步模式进行数据库操作，异步框架采用 tokio。
> Rust 版本的驱动，从项目结构上参考 Rust 下的 Postgres 驱动，该驱动源码位于 ```https://github.com/rust-postgres/rust-postgres```。
> 你可以访问本机已建好的达梦数据库实例，Host: 127.0.0.1, Port: 5236, User: SYSDBA, Password: SYSDBA, Database: SYSDBA, Table: SAMPLE
> 你可以新建表，修改你创建的表，删除你创建的表，添加、修改、删除、查询数据
> 你可以操作的目录是当前目录，也只有当前目前，不允许操作其他目录的文件。
> 任何时候都不允许修改 ```./dmPython```，这个目录对于你来说是只读的。

### 编码要求

1. 生成清晰易懂、易于维护的代码。
2. 为每个函数添加必要的注释，并为每个函数增加至少3个测试用例，以保证测试覆盖率。
3. 使用符合 sqlx 标准的测试用例，并确保测试通过，在必要时你可以自行创建表或向表内添加数据。
4. 遇到问题不要猜测，而是应当去 Python 版本的驱动代码中寻找答案。
5. 严格区分 test 和 example，test 内的函数用于测试，example 内的函数用于示例。
6. 你可以按需要自行编写其他脚本，将你编写的脚本都放置于 scripts 目录下，不要使用临时脚本。

### 项目管理要求

1. 根据上述信息，首先生成 AGENT.md 和 DESIGN.md 文件，后续的操作严格按照这两个 md 的内容来进行。
2. 在实际编码过程中，如果设计、实现等与 AGENT.md 或 DESIGN.md 中描述的流程不一致，请及时修改 md 文件。
3. 确保项目结构清晰，并使用 git 进行版本管理（目前先使用本地git），每次有效的改动都需要提交 git，并确保每次的提交都是可编译可运行的。
